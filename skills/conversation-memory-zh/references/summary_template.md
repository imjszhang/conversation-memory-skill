# 摘要模板

此模板用于生成每个对话记忆的 summary.md 文件。

## 模板内容

```markdown
# 对话记忆：{主题标题}

## 元信息

- **时间**：{YYYY-MM-DD HH:MM}
- **持续**：约 {N} 分钟
- **对话轮次**：{N} 轮
- **关键词**：{关键词1}, {关键词2}, {关键词3}

## 主题摘要

{用 1-2 段话概括这次对话的主题和背景}

## 关键决策

1. {决策1}
2. {决策2}
3. ...

## 重要结论

- {结论1}
- {结论2}
- ...

## 待办事项

- [ ] {待办1}
- [ ] {待办2}
- ...

（如无待办事项可删除此节）

## 相关文件

- `{文件路径1}` - {简要说明}
- `{文件路径2}` - {简要说明}
- ...

（如无相关文件可删除此节）

## 溯源

如需查看完整原始对话，请参阅 [conversation.md](conversation.md)
```

## 填写说明

### 主题标题

用简短的词语描述对话主题，例如：
- "记忆系统设计"
- "Kaichi 工作流重构"
- "API 接口优化"

### 关键词

**这是最重要的字段**，决定了记忆能否被正确召回。

要求：
- 提取 3-8 个关键词
- 包含主题相关的核心术语
- 便于后续搜索和匹配

示例：
```
skills, 记忆, 召回, 索引, 渐进式加载
```

### 元信息

- **时间**：对话开始时间
- **持续**：大致估算的对话时长
- **对话轮次**：用户发言次数

### 主题摘要

用自然语言描述对话的核心内容，便于快速了解这次对话讨论了什么。

### 关键决策

列出对话中做出的重要决定，使用动词开头：
- "采用 xxx 方案"
- "决定使用 xxx"
- "选择 xxx 而非 yyy"

### 重要结论

列出对话得出的结论或成果：
- 设计方案
- 问题的答案
- 达成的共识

### 待办事项

如果对话中提到了后续需要做的事情，在此记录。

### 相关文件

列出对话中涉及的代码文件、文档、配置等路径。

## 与旧版本的区别

新版 summary.md 与旧版 SKILL.md 的区别：

| 项目 | 旧版 SKILL.md | 新版 summary.md |
|------|--------------|----------------|
| YAML 前置元数据 | 有 | **无** |
| 作为独立技能 | 是 | **否** |
| 自动加载 | 元数据被加载 | **不自动加载** |
| 索引方式 | Skills 机制 | **主技能索引表** |

新版本通过主技能的索引表来管理记忆，而不是让每个记忆成为独立技能。
