# 原始对话模板

此模板用于保存完整的原始对话内容，作为溯源层。

## 模板内容

```markdown
# 原始对话记录

## 对话信息

- **开始时间**：{YYYY-MM-DD HH:MM:SS}
- **结束时间**：{YYYY-MM-DD HH:MM:SS}
- **对话轮次**：{N} 轮

---

## 对话内容

### 用户 [{HH:MM:SS}]

{用户的第一条消息}

---

### Claude [{HH:MM:SS}]

{Claude 的第一条回复}

---

### 用户 [{HH:MM:SS}]

{用户的第二条消息}

---

### Claude [{HH:MM:SS}]

{Claude 的第二条回复}

---

（继续记录所有对话轮次...）
```

## 填写说明

### 对话信息

- **开始时间**：对话第一条消息的时间
- **结束时间**：对话最后一条消息的时间
- **对话轮次**：用户发言的总次数

### 对话内容

1. **保留完整内容**：不要省略或概括，保留原始对话的完整内容
2. **标注发言者**：使用 `### 用户` 和 `### Claude` 区分
3. **标注时间**：如果有时间信息，在方括号中标注
4. **使用分隔线**：每轮对话之间用 `---` 分隔，便于阅读

### 特殊内容处理

#### 代码块

保留原始代码块格式：

```markdown
### Claude [{HH:MM:SS}]

这是一个示例代码：

\`\`\`javascript
function hello() {
  console.log("Hello, World!");
}
\`\`\`
```

#### 长内容

如果某条消息特别长（如大段代码），可以完整保留，不要截断。

#### 图片/文件引用

如果对话中涉及图片或文件，记录其路径或描述：

```markdown
### 用户 [{HH:MM:SS}]

[附件：screenshot.png - 系统架构图]

请帮我分析这个架构...
```

## 为什么需要原始对话

1. **溯源**：摘要可能遗漏细节，原始对话是最准确的记录
2. **上下文**：理解决策的背景和讨论过程
3. **搜索**：可以在原始对话中搜索特定内容
4. **学习**：回顾之前的讨论方式和思路
